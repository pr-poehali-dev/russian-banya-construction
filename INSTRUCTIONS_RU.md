# Инструкция по поиску удаленных проектов в Git истории

## Проблема
Проекты "Красная слудка" и "Ключи" были удалены из файла `src/components/home/projectData.ts` при последней декомпозиции. Нужно восстановить их полные данные из git истории.

## Решение

Создано несколько скриптов для поиска этих проектов в истории git.

### Вариант 1: Bash скрипт (рекомендуется)

Запустите скрипт `find_projects.sh`:

```bash
chmod +x find_projects.sh
./find_projects.sh
```

#### Что делает скрипт:
1. Проходит по всем коммитам, которые изменяли файл `projectData.ts`
2. Ищет упоминания "Красная" и "Ключи" в каждой версии файла
3. Сохраняет найденные версии файлов в директорию `git_history_results/`
4. Создает файлы с извлечениями (grep выдержки) для быстрого просмотра

#### Результат:
Все результаты будут в папке `git_history_results/`:
- `krasnaya_XXXXXXXX.ts` - полная версия файла с проектом "Красная слудка"
- `krasnaya_XXXXXXXX.extract.txt` - извлечение с контекстом
- `klyuchi_XXXXXXXX.ts` - полная версия файла с проектом "Ключи"
- `klyuchi_XXXXXXXX.extract.txt` - извлечение с контекстом
- `commit_XXXXXXXX_info.txt` - информация о коммите

### Вариант 2: Python скрипт

Запустите Python скрипт:

```bash
python3 extract_projects.py
```

#### Что делает скрипт:
1. Более детальный анализ с попыткой извлечь структурированные данные
2. Пытается распарсить TypeScript и извлечь поля проекта
3. Сохраняет результаты в файлы с именами проектов

### Вариант 3: Простой bash скрипт

```bash
chmod +x show_history.sh
./show_history.sh
```

Выводит на экран все версии файла, где найдены искомые проекты.

## Что искать в результатах

Для каждого проекта нужно извлечь:

```typescript
{
  img: "URL_главного_фото",
  title: "Название проекта",
  size: "Размер (например: 6×8 м)",
  material: "Материал (например: Профилированный брус)",
  location: "Локация (например: д. Красная Слудка, Пермский край)"
}
```

И массив фотографий для галереи из `projectGalleries`:

```typescript
[
  "URL_фото_1",
  "URL_фото_2",
  // ...
]
```

## Ручной поиск (если скрипты не работают)

Если скрипты по какой-то причине не работают, можно искать вручную:

```bash
# Получить список всех коммитов файла
git log --oneline -- src/components/home/projectData.ts

# Посмотреть содержимое файла в конкретном коммите
git show COMMIT_HASH:src/components/home/projectData.ts

# Искать в истории
git log -p -S"Красная" -- src/components/home/projectData.ts
git log -p -S"Ключи" -- src/components/home/projectData.ts
```

## Восстановление проектов

После того как данные найдены:

1. Откройте найденные файлы из `git_history_results/`
2. Найдите блоки с проектами "Красная слудка" и "Ключи"
3. Скопируйте данные проекта (title, size, material, location, img)
4. Найдите соответствующий массив в `projectGalleries`
5. Добавьте проекты обратно в `src/components/home/projectData.ts`

## Примечания

- Все скрипты безопасны - они только читают git историю, не изменяют файлы
- Результаты сохраняются в отдельную директорию
- Можно запустить несколько скриптов для перекрестной проверки
